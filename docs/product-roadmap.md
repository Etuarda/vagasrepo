# Product Roadmap

## Current value proposition

Help developers manage applications, keep professional data structured and generate deterministic ATS-friendly resumes based only on real profile evidence.

## MVP focus

Keep these flows stable before adding new scope:

- account and session;
- global profile;
- subprofiles with plan limits;
- deterministic matching;
- optimized resume generation;
- matching history and recalculation;
- applications tracking;
- shared jobs with summarized match;
- subscription billing.

## Metrics to track

- profile completion rate;
- matching analyses created per user;
- optimized resumes downloaded;
- applications created from matching;
- shared jobs viewed;
- conversion from free to paid;
- plan limit hits;
- billing failures and webhook retries;
- incomplete analysis rate.

## Do not add now

- generative AI text rewriting;
- automatic job scraping;
- microservices;
- complex orchestration platforms;
- PDF auto-import into profile without manual review;
- Google Docs OAuth before the core resume/history flow is stable.

## Next product bets

1. Improve analysis explainability in history.
2. Add profile completion checklist by section.
3. Add comparison modal for global profile versus subprofiles.
4. Move heavy file storage to object storage if PDF volume grows.
5. Add e2e browser tests for login, matching, billing sandbox and application tracking.
